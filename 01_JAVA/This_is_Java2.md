# 예외처리

- 에러가 생겼을 때 프로그램이 동작을 하지 않게 되는데 잘못된 조작 또는 코딩으로 인해 예외가 발생했을 경우 예외처리를 통해 프로그램이 계속 유지되도록 해준다



## 실행예외

컴파일러가 체크를 하지 않는다 <- 개발자에 의해서 예외 처리 코드 삽입



### NullPointerException

객체 참조가 없는 상태

### ArrayIndexOutOfBoundsException

배열에서 인덱스 범위를 초과하여 사용하는 경우

### NumberFormatException

문자를 숫자로 변환할 때 숫자로 변환할 수 없는 문자가 있는 경우

### ClassCastException

클래스 변환을 할 수 없을 때

## 예외처리

프로그램이 종료하지 않고 계속 실행되게 해준다

### 다중 catch

예외의 종류가 다양할 때 각각의 예외들을 처리할 때 사용
